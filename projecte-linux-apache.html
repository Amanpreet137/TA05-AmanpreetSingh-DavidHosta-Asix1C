Servidor Linux amb Apache, SSL i còpies de seguretat

Projecte d’administració de sistemes orientat a desplegar un servidor web segur, protegint l’accés a recursos i garantint recuperació de dades mitjançant backups automatitzats.

Tornar a l’inici Veure tots els projectes

Fitxa tècnica

Rol: Administració i desplegament

Entorn: Ubuntu Server (VM / laboratori)

Àrees: Web, seguretat, permisos, backups

Eines: Apache2, OpenSSL, .htaccess, Duplicati, SSH

Detalls del projecte

Objectius

  • Publicar un servei web estable amb Apache i Virtual Hosts
  • Xifrar comunicacions amb HTTPS (SSL/TLS)
  • Restringir accés a directoris amb autenticació bàsica
  • Automatitzar còpies de seguretat amb versionat i restauració

Implementació (resum)

  • Configuració de Virtual Hosts i estructura de directoris
  • Activació de mòduls necessaris (ssl, headers, etc.)
  • Certificat SSL (autofirmat) i redirecció HTTP → HTTPS
  • .htaccess + fitxer d’usuaris i proves d’accés
  • Duplicati amb planificació i verificació de restauració

Resultats

El servidor queda operatiu amb navegació xifrada, control d’accés i un sistema de backups que permet recuperar informació després d’una incidència. La solució és escalable per afegir més llocs web i polítiques de seguretat.

Millores futures

  • Certificat Let’s Encrypt i renovació automàtica
  • Hardening d’Apache (headers, limitació mètodes, rate limit)
  • Monitorització bàsica (logs + alertes)
  • Backups offsite (núvol o servidor remot)
